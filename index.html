[雲原生軟體開發]TDD KATA POTTER

發表於 2022-05-15 更新於 2022-05-27 分類於 Cloud ， 雲原生軟體開發 Disqus：

¶問題描述

有一家書店在賣哈利波特書籍系列，每一本書定價 $8 歐元。買兩本不同的書可以打 5% 的折扣、買三本不同的書可以打 10% 的折扣、買四本不同的書可以打 20%。如果買到五本可以打到 25% 的折扣。
請注意，例如，如果您購買了四本書，其中 3 本書是不同的書名，那麼您可以在其中的 3 本書上獲得 10% 的折扣，但第四本書的價格仍然是 8 歐元。
任務是寫一段 Code 來計算任何可以想像到的購物籃的價格，並提供盡可能大的折扣。

¶案例

例如：以下總價是多少錢？
第一本書 : 2 本
第二本書 : 2 本
第三本書 : 2 本
第四本書 : 1 本
第五本書 : 1 本

Ans:

它不是 5 _ 8 _ 0.75 + 3 _ 8 _ 0.90。事實上 4 _ 8 _ 0.8 + 4 _ 8 _ 0.8 是更便宜。所以這個 Kata 的訣竅不在於你接受的驗收測試是錯誤的。訣竅在於，您必須編寫一些足夠智能的代碼，才能注意到兩套四本書比一套五本書和一套三本書便宜。

發表於 2022-05-14 更新於 2022-05-27 分類於 Cloud ， 雲原生軟體開發 Disqus：

這篇文章主要介紹在軟體開發流程裡面的 TESTING，從 TESTING 種類、介紹 TDD 與實作部分，以保齡球計分為例實作開發環境測試與 TDD 練😀

¶TESTING 種類

顆粒度由小到大排列

  1. Unit Test(單元測試)
  2. Intefration Test(整合測試)
  3. End to End Testing(E2E 測試)
  4. Acceptance testing(驗收測試)
  5. Performance testing

同一目的：確保系統品質與穩定性

下圖根據測試速度與相依性進行排序，越底層代表越獨立，可以看到手動測試所花費的成本跟時間最高，不過其可以涵蓋的範圍最大。

¶TDD (Test-Driven Development)

測試驅動開發（Test-Driven Development,簡稱 TDD）是一種程式開發的技巧，簡單來說就是先寫測試程式，然後才實作功能，具體的流程則是透過 Red-Green–Refactor cycle 的循環。

思考問題：當想用 TDD 時，需求不明怎麼寫測試？？
理想狀況

  • 那為什麼還會出現 TDD？
    理想情況，TDD 的出發點是 User Acceptance criteria，當測試訂下來後，當我的程式有通過，表示有符合使用者需求與達成使用。

其他延伸：ATDD(Acceptance TDD),BDD(Behavior-driven development)

IEEE-CIS Credit fraud detection

¶1.動機 Motivations

小心網購！今年信用卡小額網路盜刷大增 26.5％
今年前 5 月信用卡盜刷高達 6.2 億，多數來自蘋果、Google、亞馬遜

針對時事新聞，我有興趣建立一個信用卡檢測系統來識別欺詐性信用卡交易。這樣客戶就不會因為他們沒有購買的商品而被收取費用。

這個專案的好處:

  • 有效且不費力地打擊欺詐
  • 防止狡猾的欺詐交易(手動定義的規則通常無法預料)
  • 為全球數百萬人提高欺詐交易警報的效力，幫助數十萬企業減少欺詐損失並增加收入

¶2.資料視覺化 Exploratory data analysis (EDA)

Data Source link: IEEE-CIS Fraud Detection

Sentimental Analysis On Amazon’s Alexa Reviews

發表於 2022-03-27 更新於 2022-05-27 分類於 Data Analytics ， Machine Learning Disqus：

¶Summary

情感分析（Sentiment Analysis）是挖掘文本數據（如評論、陳述）並對其背後的情緒進行預測之過程。隨著電子商務發展漸趨成熟且相關資料的蒐集越加容易，現今不少企業開始採用情感分析來監測自家產品受歡迎的程度，其中「顧客的回饋」，如文字評論、產品評分等等，是企業進行情感分析之關鍵。
此報告期望透過程式實作，對 Amazon Alexa（亞馬遜公司推出的一款智慧型助理）這項產品之顧客回饋進行深入的情感分析。本組採用的模型包含：樸素貝式分類器（Naive Bayes Classifier）、邏輯斯迴歸（Logistic Regression）、決策樹（Decision Tree）、隨機森林（Random Forest）、XGBoost、長短期記憶模型（LSTM），進一步比較各模型之預測準度(Accuracy)、精確度(Precision)、召回率(Recall)、F1-Score 以及 ROC Curve 下的面積(AUC)。

關鍵字：Sentiment Analysis、Natural Language Processing、Machine Learning、Deep Learning、Unbalanced Dataset、Amazon Alexa

¶動機

現今，許多電子商務企業、社群網站，如亞馬遜的顧客回饋、各種實體店家的 Google 評論等，生成了大量「以評論形式」出現的數據。若手動審查所有文字評論將會是一項耗時且繁瑣的任務。因此，設計自動情感分析的機器學習演算法，將可使買賣雙方甚至潛在顧客，都能對自己的決策有更精確的參考依據。
機器學習算法有助於從大量的數據中提取有用信息並幫助做出決策。本組期透過機器學習算法，自動分析顧客對 Amazon Alexa 及相關產品的評論與回饋，並將其分類為正面或負面。如此一來，本組便可快速且有效率的對大量數據進行處理，並判斷評論者對實體產品的情感和看法。

¶資料集敘述

本組自 Kaggle 取得 Amazon Alexa 相關產品的顧客回饋資料集。首先，本組對原始資料集進行探索式資料分析(Exploratory Data Analysis，簡稱 EDA)，觀察到此資料集包含 5 種屬性共 3150 筆資料，且無任何缺失值。
屬性資料

  • Rating：評價，1~5 顆星。
  • Date：日期。
  • Variation：產品型號。
  • Verified_Reviews：顧客評論，以文字形式呈現。
  • Feedback：回饋，1 為正面回饋；0 為負面回饋。

¶3.1 Feedback & Rating 的處理

由於預測模型的主要目的在於判斷顧客評論會被歸類在正面回饋或負面回饋，即 Feedback 屬性是 1 還是 0，因此先觀察此屬性的原始分佈。

資管所推甄心得 面試篇

發表於 2021-06-28 更新於 2022-05-27 分類於 人生旅途 ， 推甄 Disqus：

又到了研究所推甄的季節了，就來分享一下我準備程序與當時面試時的題目與網站推薦，希望可以幫助到大家☺️

¶背景介紹

  • 國立彰化師範大學資管系(財務金融技術學系輔修)
  • 在校成績 : 系排 1%
  • 英文多益成績 : 775
  • 課外活動 : 科技部大專生研究計畫、社會責任團隊營運長、大學迎新總召
  • 推甄結果 : 成大(正取)、中央(正取)、政大(正取)、交大(備五，後來備上)、中山(逕行錄取)

¶面試準備心法

  • 先求有再求好:和備審篇說得相同，寫不出來時先寫一點，慢慢再把自己的東西優化。

  • 要有邏輯框架: 因為面試時會很緊張，可以在你準備題目(介紹研究計畫、專題等等)時，有一定的邏輯框架，或是問你原因(你為甚麼要輔修?)時，你可以列點說明，好處是你容易記，在緊張時不容易忘，且呈現方式比較順暢。

¶個人面試準備程序

¶1.蒐集面試題目

我在做備審疲憊時或是閒暇時，會先爬文蒐集相關面試題目，當然越多越好，近年來的題目先準備。
大致根據自己將要準備的題目分類，但建議還是要標記一下學校，這樣你在面試學校前一天，可以多加複習特定學校的題目。

¶2.自我介紹 (3版本) : 1分30秒 | 1分鐘 | 30秒

¶前言: 判斷要不要推甄 三個重要因素

  1. 校名與在校成績
  2. 科技部
  3. 英文

從大一開始，教授就特別強調這三項，經過這次實際推甄，真的覺得這三項佔很大的因素，尤其是在校成績，如果成績不夠可能連面試的機會都沒有。

其他兩項若擁有其中一項，都是會給你帶來加分的，所以我建議若在校成績不算太理想的，還是可以試試看，應該還是有機會，但最好同時準備一下考試。

📓 詢問自己學校學長姐經驗來判斷是否要推甄，這種方式來判斷也很準確呦~~
