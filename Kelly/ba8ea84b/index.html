這篇文章主要介紹在軟體開發流程裡面的 TESTING，從 TESTING 種類、介紹 TDD 與實作部分，以保齡球計分為例實作開發環境測試與 TDD 練😀

¶TESTING 種類

顆粒度由小到大排列

  1. Unit Test(單元測試)
  2. Intefration Test(整合測試)
  3. End to End Testing(E2E 測試)
  4. Acceptance testing(驗收測試)
  5. Performance testing

同一目的：確保系統品質與穩定性

下圖根據測試速度與相依性進行排序，越底層代表越獨立，可以看到手動測試所花費的成本跟時間最高，不過其可以涵蓋的範圍最大。

¶TDD (Test-Driven Development)

測試驅動開發（Test-Driven Development,簡稱 TDD）是一種程式開發的技巧，簡單來說就是先寫測試程式，然後才實作功能，具體的流程則是透過 Red-Green–Refactor cycle 的循環。

思考問題：當想用 TDD 時，需求不明怎麼寫測試？？
理想狀況

  • 那為什麼還會出現 TDD？
    理想情況，TDD 的出發點是 User Acceptance criteria，當測試訂下來後，當我的程式有通過，表示有符合使用者需求與達成使用。

其他延伸：ATDD(Acceptance TDD),BDD(Behavior-driven development)

¶TDD 流程

Red - 先寫測試程式，此時因為還沒有實作功能，所以這個測試是會失敗
Green – 快速實作出功能，讓上述的測試通過
Refactor – 重構程式碼，去除多餘重複的代碼，提高程式碼品質，同時維持測試通過

¶建立開發環境

  1. 安裝 node.js LTS 版本
  2. 開啟 terminal 並進入到一個可以建立專案的資料夾
  3. 執行 npx create-nx-workspace 指令
  4. workspace 輸入 w12-testing
  5. 選擇建立 angular
  6. project 輸入 bowling
  7. Application Name 輸入 bowling
  8. Default stylesheet format 選擇 CSS
  9. Use Nx Cloud 選擇 No
  10. 就讓他跑一下，過程中會產生預設專案檔案及安裝 npm package
  11. 進入 w12-testing 資料夾，輸入 code . 或是打開 VSCode 並開啟該資料夾

  1. 驗證專案可不可以正常跑起來
    npx nx serve

  2. 專案可不可以正常 build
